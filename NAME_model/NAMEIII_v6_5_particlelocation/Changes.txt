07/01/2011

Ticket #32: Upgrade GRIB support in NAME to use the ECMWF GRIB_API library package.

Addition of GRIB_API software to NAME library (including some additional local files
and directories).
   
   a) Added a new directory /Code_GribAPI into the NAME directory structure
      containing GRIB_API source files.

      for the compiled GRIB_API libraries, etc.

16/12/2011

Ticket #55: Python Code

Added a directory /Code_Python into the NAME directory structure to store python code.
Currently only contains code for converting NAME field output files to NetCDF.

Susan Leadbetter

!-------------------------------------------------------------------------------------
31/01/2012

Ticket #63: Add a User Guides directory and installation test suite.

Added a new directory /User_Guides into the NAME directory structure to store
user guide documentation and a basic test suite for verifying the NAME installation.

This directory contains two subdirectories

 - User_Guides/Docs (intended for the storage of user guide documentation), and
 - User_Guides/Test_Runs (for the installation test suite).

A collection of files and folders have been added for running an installation test
on Linux and Windows systems.

Andrew Jones

!-------------------------------------------------------------------------------------
17/02/2012

Ticket #34: Restructure MakeExternalUsersVersion code to isolate external software in
            separate top-level directories.

Added a new directory /MakeExternalUsersVersion in the top-level NAME structure
to store files involved in the creation of external code versions. The existing
build script MakeExternalUsersVersion.sh has been moved into this new subdirectory.

Andrew Jones

!-------------------------------------------------------------------------------------

17/02/12

Ticket #64: Activities associated with freezing of version 6.1.

Added Windows Visual Studio project file NameIII.sln for use in compiling NAME
with the Intel Fortran compiler on Windows systems. Eike Mueller had previously
produced this file for version 6.0 (see ticket #16: Porting to Windows)
but it was not copied into the main code trunk.

Updated the Compaq Visual Studio file NameIII.opt for use in compiling NAME with
the Compaq compiler on Windows (as used by David Thomson).

Andrew Jones
