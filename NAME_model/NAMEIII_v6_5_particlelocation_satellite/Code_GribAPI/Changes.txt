14/12/10
--------

The ECMWF GRIB API is an application program interface accessible from C and FORTRAN programs
developed for encoding and decoding WMO FM-92 GRIB edition 1 and edition 2 messages.

Version 1.9.5 downloaded (see http://www.ecmwf.int/products/data/software/download/grib_api.html)
and added to NAME III.

The configuration script supplied with the software package will automatically determine the
operating system, available compilers, etc. and generate an appropriate makefile for the
compilation step.

Added a standard compile script LinuxIntelRelease to simplify the compilation/installation process.
This includes an additional step to build/install shared object library code (.so files), as the
supplied package only creates static library code (.a files). The additional step is required to
comply with IPR issues concerning the linking of external software into Met Office code.
Dynamic linking at runtime also produces a smaller NAME executable file. '_64bit' is added
to the names of the libraries when compiled on 64-bit systems.

Added local definition files describing the parameters in our ECMWF ERA datasets for NAME.
The parameter descriptions are contained in the four files shortName.def, paramId.def, units.def
and name.def in .../Code_GribAPI/local/definitions/grib1/localConcepts/egrr (here 'egrr' is the
identifier referring to UK Met Office local definitions). The entire local definitions directory
.../Code_GribAPI/local is copied to .../SharedLibraries_Linux/local during the installation process.
The environment variable GRIB_DEFINITION_PATH can be set to use local definition files in preference
to the definition files provided within the distribution. To do this, set the environment variable
GRIB_DEFINITION_PATH=${SHAREDLIB_DIR}/local/definitions:${SHAREDLIB_DIR}/share/definitions
where ${SHAREDLIB_DIR} is the top-level directory of the GRIB API installation.

Andrew Jones

18/05/11
--------

Ticket #46: Revised NAME met definitions for ECMWF forecast data at IFS cycle 37r2.

Updated local definition files describing processed parameters in our ECMWF datasets for NAME.
From 'Mk2' onwards, all processed fields in the real-time forecast files (i.e. OPER and EPS)
have their GRIB keys modified to correctly identify the field contents. This was already
being done for reanalysis datasets (ERA40 and ERAInt). The information on 'local parameter
definitions' in .../Code_GribAPI/local/definitions/grib1/localConcepts/egrr has been updated
to include these extra parameters. Similarly, these local parameters have been included in
a new local copy of the ECMWF definition files, which are stored in the new folder
.../Code_GribAPI/local/definitions/grib1/localConcepts/ecmf (here 'ecmf' is the identifier
referring to ECMWF local definitions). This is necessary because the originating centre
cannot be modified for EPS fields (as this loses the ensemble number information stored
in a specific local extension for ECMWF data), and therefore the new parameters need to be
defined for originating centre ECMWF as well as UK Met Office.

Andrew Jones

28/10/2014

Ticket #109: Add Mk3 met definition files for real-time ECMWF forecast data.

Updated local definition files describing processed parameters in our ECMWF datasets
for NAME. In the real-time forecast files for the 'Mk3' met data feeds, the parameter
"total precipitation" has been replaced by "total rainfall" as use of this parameter
is more consistent for modelling wet deposition under the PC2 changes.

Entries for the additional parameter "mean total rainfall rate since last output" have
been added into the four files shortName.def, paramId.def, units.def and name.def in
the two subfolders

Code_GribAPI/local/definitions/grib1/localConcepts/egrr
Code_GribAPI/local/definitions/grib1/localConcepts/ecmf

where 'egrr' is the identifier referring to UK Met Office local definitions, and
'ecmf' is the identifier referring to ECMWF local definitions.

Andrew Jones
