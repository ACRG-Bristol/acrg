Changes prior to version 4.0 not recorded.

!-------------------------------------------------------------------------------------

31/10/06 

NAG / HP scripts removed. Fortran preprocessor symbols NAGHPCompiler and HPHPCompiler
removed, with NAGHPCompiler replaced by IntelLinCompiler (previously one needed 
NAGHPCompiler set as well as IntelLinCompiler when running on Linux with the Intel
compiler).

Scripts LinuxIntelRelease and LinuxIntelDebug updated to use version 9 of the 
compiler.

Fortran preprocessor symbol Linux64Intel renamed LinuxIntelV9.

The compile option -Wp,-macro=no has been added to the Linux Intel compiler scripts. 
This ensures fpp doesn't do macro expansion outside of pre-processor lines (the -Wp 
bit passes what follows to fpp; the -macro=no bit seems undocumented by Intel but is
what worked with the NAG compiler). 

Fortran preprocessor symbol ExtraChecks added to the Windows Compaq compiler treatment 
(release version) for consistency with the LinuxIntelRelease script.

Dave Thomson

31/07/2015

Ticket #134: Activities associated with freezing of version 6.5.

Name2Adms.vfproj
Updated the Visual Studio project file for compiling with the Intel Fortran compiler
under Windows. 64-bit configurations have been added to the existing 32-bit options.
The folder configuration has also been changed to use separate "Win32" and "x64"
compilation directories (each with subdirectories "Release" and "Debug"). The names
of the executables in /Executables_Win have also been revised for consistency with
the Linux filename convention (e.g., use of "_64bit" for 64-bit executables).

Also revised the compiler directives, replacing the directive "CompaqPCCompiler"
with "IntelLinCompiler,IntelWin" (as had been previously done for NAME itself).

Andrew Jones
